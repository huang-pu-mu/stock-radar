# Stock Radar V1.2-3 NAS Docker 排程補丁

## 一、本次目標

建立一個只負責排程匯入資料的 Node.js Docker 容器。

這個容器不開放網站、不開放 API、不需要 HTTPS、不需要憑證。

流程：

```text
NAS Docker Scheduler
→ 每天 18:30 執行 npm run scheduled:daily
→ 依序執行 npm run daily 與 npm run official:daily
→ 寫入 NAS MariaDB
→ Vercel 前端 / API 照常讀資料
```

## 四、本機先測試

如果你的本機有 Docker Desktop，可以先測：

```powershell
npm run docker:scheduler:build
npm run docker:scheduler:up
npm run docker:scheduler:logs
```

手動觸發一次排程：

```powershell
npm run docker:scheduler:test
```

關閉容器：

```powershell
npm run docker:scheduler:down
```

## 五、NAS 上的注意事項

### 1. .env 的 DB_HOST 不要用 127.0.0.1

Docker 容器內的 127.0.0.1 是容器自己，不是 NAS 主機。

建議使用其中一種：

```env
DB_HOST=NAS區網IP
```

或：

```env
DB_HOST=host.docker.internal
```

或如果 MariaDB 也是 Docker 容器，且在同一個 Docker network：

```env
DB_HOST=MariaDB容器名稱
```

### 2. 此容器不需要對外 port

`docker-compose.scheduler.yml` 沒有 ports 設定。

代表它不提供網站，也不提供 API，因此不需要 HTTPS 或憑證。

### 3. 排程時間

預設每天 18:30 執行。

如果要修改時間，改這個檔案：

```text
stock-radar-api/docker/scheduler-crontab
```

目前內容：

```cron
30 18 * * * /bin/sh /app/docker/run-scheduled-daily.sh >> /app/logs/cron.log 2>&1
```
